# Security

Please do not report secrets, credentials or confidential matter materials in public issues, pull requests or discussions.

If a credential is exposed in a local copy, revoke or rotate it immediately, then remove the exposed material from the relevant history. Do not bypass GitHub secret-protection blocks without first confirming that the detected value is not sensitive.

For a potential repository or workflow vulnerability, use GitHub private vulnerability reporting when it is enabled, or contact the repository maintainer through a private channel. Do not open a public issue for sensitive details.

Legal OS is a pre-release workflow framework. It is not a production security boundary and does not replace the security, privacy, access-control or review requirements of the system in which it is used.
