# Legal OS Matter Memory Workspace

This is a generic, source-locked workflow for keeping matter memory lean and useful. It does not turn memory into current legal authority, current fact, or an automatic global rule.

## Classification

- L1: durable cross-task core rules.
- L2: stable specialized workstream rules.
- L3: repeatable Skills, templates, and workflows.
- L4: matter or project clues, used only as historical context.
- L5: dynamic amounts, dates, progress, procedural status, and other facts that require a dated source and refresh trigger.
- L6: temporary drafts, guesses, logs, paths, and unconfirmed context that should not be retained.

## Operating method

Capture the smallest source-linked item, classify it, search for duplicates, minimize sensitive content, record verification and expiry, and choose keep, merge, downgrade, convert, archive, or skip. Load memory by trigger: current materials first, the named matter next, and only the rules or Skills needed for the task.

## Hard stops

Do not put case-specific facts in global rules or public Skills. Do not treat an unverified amount, date, status, or legal conclusion as current. Pause when privacy, scope, retention, source, version, or user authorization is unclear. Long-term asset changes require a candidate diff, preserved rollback, and explicit approval.

Reusable observations and corrections route to the learning-governance workflow; substantive legal work routes to its primary workflow; file delivery routes to the file-delivery workflow.
